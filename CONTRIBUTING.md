# Правила внесения изменений

## Главный файл

`knowledge/kupavna_master_knowledge_base.md`

## Перед изменением

Проверь, что данные не противоречат сайту `https://kupavna.pro/` и текущей логике базы.

## Формат изменения

В Pull Request укажи:

- что изменилось;
- источник данных;
- какие разделы затронуты;
- есть ли риск устаревания;
- нужно ли обновить exports.

## Статусы данных

- `подтверждено` — можно использовать как факт;
- `требует проверки` — нельзя использовать в КП как факт;
- `рабочий материал` — можно использовать как сырьё;
- `удалено / не использовать` — запрещено к использованию.
