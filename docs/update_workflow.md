# Процесс обновления базы

## Главное правило

Факты по объекту проверяются через сайт `https://kupavna.pro/` или через подтверждённые документы / брокера / собственника.

## Как обновлять

1. Найти новое или изменившееся утверждение.
2. Проверить источник.
3. Определить статус:
   - подтверждено;
   - требует проверки;
   - рабочая гипотеза;
   - устарело / удалить.
4. Обновить `knowledge/kupavna_master_knowledge_base.md`.
5. Добавить запись в `changelog.md`.
6. Если изменение спорное, вынести его в раздел конфликтов, а не подменять факт.

## Что нельзя делать

- Добавлять коммерческие условия без даты и источника.
- Предлагать проданные блоки 8 и 10.
- Презентовать аренду как активный продукт.
- Продавать прогнозную доходность как гарантию.
- Использовать старые материалы как источник истины без проверки.
